## Assigned Plan

Eliminate shared memory bank conflicts and increase XTILE: keep Real as double, but change 'Real sm_psi[4][BSIZE][BSIZE]' to 'Real sm_psi[4][BSIZE][BSIZE+1]'. With BSIZE=16 and doubles, rows are 128 bytes = 32 banks, causing 2-way conflicts when a warp spans two consecutive tjj rows; adding +1 double per row shifts row 1 by 2 banks and eliminates conflicts. In the same commit, increase XTILE from 20 to 32: this makes each thread block perform more x-direction iterations, amortizing the 4 __syncthreads() calls per loop over more useful work, reducing the effective fraction of time spent at barriers. Update the grid-dimension formula accordingly (bdimx = (nx-2)/XTILE + ...). No correctness risk since algorithm is unchanged. Careful: verify nLast_x boundary logic still holds at the last block; test with nx not a multiple of XTILE. This combination directly targets both the bank conflict issue and partially the 47% barrier stall.
