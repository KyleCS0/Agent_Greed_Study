## Assigned Plan

Increase thread-level memory-latency hiding by raising the block size and tuning occupancy. Currently BLOCK_SIZE=256 with 16 registers/thread gives ~80% occupancy on sm_89; however the 59% memory/long-scoreboard stall rate suggests more in-flight warps would help. Try BLOCK_SIZE=512 or BLOCK_SIZE=1024 to increase warps-per-SM and better hide DRAM latency. Additionally, add `__launch_bounds__(BLOCK_SIZE, 2)` to the kernel to give the compiler a hint to limit register pressure and allow more concurrent blocks. Also annotate the two global loads with `__ldg()` (read-only cache via texture path) since d_input is only written by the same kernel in a previous launch: `int ai = __ldg(&a[i]); int asw = __ldg(&a[swapped_ele]);`. This routes loads through the 128-byte read-only cache, improving effective bandwidth for repeated accesses to the same cache lines across warps. Correctness risk: `__ldg` is only safe if the loaded locations are not being written by threads in the same kernel launch—confirm that each launch only writes to non-overlapping pairs.
