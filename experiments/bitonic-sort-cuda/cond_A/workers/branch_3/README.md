## Assigned Plan

Reduce kernel launch overhead and CPU-side `pow()` cost by replacing the double nested host loop (300 launches for n=24) with a single persistent-kernel or batched approach for the global stages. Concretely: (a) Replace all `pow(2, x)` calls in the host loop with integer bit-shifts (`1 << x`) to eliminate FP-to-int conversion. (b) Pass seq_len and two_power as compile-time template parameters for the most common values, or pack both into a single 64-bit argument to reduce kernel argument overhead. (c) Alternatively, encode the full sequence of (seq_len, two_power) pairs into a small device-side lookup table and launch a single grid-stride kernel that iterates over the table with `__syncthreads()`-equivalent barriers via cooperative groups (`grid.sync()`), requiring `cudaLaunchCooperativeKernel`. This cuts 300 kernel launches to 1. Correctness risk with cooperative groups: the entire grid must fit in the resident block count of the device; query `cudaOccupancyMaxActiveBlocksPerMultiprocessor` and size the grid accordingly. Without cooperative groups, at minimum apply (a) and (b) to reduce per-launch overhead.
