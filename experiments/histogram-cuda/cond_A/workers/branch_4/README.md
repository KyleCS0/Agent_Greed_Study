## Assigned Plan

Eliminate the cudaMalloc/cudaFree of d_part_hist inside the timed region by pre-allocating it once outside the timing loop. In run_smem_atomics (and run_gmem_atomics), the partial histogram buffer is allocated and freed every call. For the benchmark's 100-iteration timing loop this adds cudaMalloc latency inside each measured call. Refactor run_smem_atomics to accept a pre-allocated d_part_hist pointer, or allocate it once before the timing loop in the caller (TestMethods / RunTest). The GpuTimer wraps both kernel launches but also the cudaMalloc call implicitly synchronizes, inflating measured time. Move the allocation outside the timer. Correctness risk: none if the buffer is zeroed properly before each kernel launch (the smem kernel zeros smem at startup, so the partial buffer only needs to be sized correctly). Performance risk: none; this is an overhead reduction.
